# 우리 회사에서 기술블로그 홈페이지를 만들건데요

우선 티사이언티픽 [티사이언티픽](https://www.tscientific.co.kr/) 홈페이지 보고 어울릴 수 있는 기술 블로그나 커뮤니티 홈페이지 어떻게 디자인해야할지 트렌드한 방향을 찾아주세요

티사이언티픽 메인 사이트 톤(신뢰·엔터프라이즈·보안)을 유지하면서, 기술블로그/커뮤니티는 좀 더 개방적이고 실무·개발자 친화적으로 가는 게 좋습니다.[^1][^2]

## 1. 티사이언티픽 브랜드 톤 정리

- 키워드: 디지털 신뢰, 정보보호·보안, 시스템통합, 클라우드, 운영까지 책임지는 파트너.[^2][^1]
- 전체 톤: B2G/B2B 엔터프라이즈, 다소 포멀하고 안정감 있는 레이아웃/컬러.
→ 기술블로그는 이 톤을 깨지 말고, “조금 더 가벼운 R\&D/엔지니어 조직의 얼굴”로 디자인하는 방향이 맞습니다.


## 2. 2025 웹·블로그 디자인 트렌드 핵심 포인트

- 미니멀 레이아웃 + 강한 타이포: 불필요한 장식 최소화, 헤드라인에 굵은 서체/큰 사이즈 사용.[^3][^4]
- 다크/로우 에너지 모드: 눈부심이 덜한 톤다운 다크, 높은 명도 포인트 컬러 조합이 확산.[^5][^4]
- 카드/벤토 그리드: 여러 글·카테고리를 카드형 또는 벤토 박스 스타일로 배열해 가독성과 스캔 속도 ↑.[^4][^5]
- 인터랙티브 요소: 호버 애니메이션, 스크롤 트랜지션, 간단한 모션으로 “정적인 SI 기업 블로그” 느낌 탈피.[^3][^4]
- 아날로그+AI 감성: 코드/모델 다이어그램 같은 디지털 요소에 메모, 형광펜, 러프한 라인 등을 살짝 섞는 트렌드.[^6][^5]


## 3. 기술 블로그 IA/레이아웃 방향

### 상단 구조

- 상단 헤더:
    - 좌측: 로고(또는 “Tech Blog by T Scientific”)
    - 우측: 메뉴 – Articles, Tech Note, Security Lab, Event/Community, About Team
- 히어로 섹션:
    - 큰 헤드라인: “디지털 신뢰를 만드는 기술 인사이트”처럼 회사 미션과 연결.[^1]
    - 서브카피: “보안·클라우드·데이터 기술 실무 노하우 공유”
    - 최신 3개 포스트를 가로 카드(벤토 스타일)로 배치.


### 콘텐츠 카드 디자인

- 썸네일: 코드 스니펫, 아키텍처 다이어그램, 보안 콘셉트 일러스트 등 일관된 스타일.
- 메타 정보: 카테고리(예: PII Detection, SIEM, Cloud, DevOps), 작성자, 태그, 예상 읽기 시간.
- 리스트 뷰: 데스크톱에서는 3열 그리드, 태블릿 2열, 모바일 1열의 반응형.


## 4. 커뮤니티/Dev 섹션 디자인 방향

- Developer/Community Hub:
    - 문서, 가이드, 샘플 코드, API 레퍼런스를 모은 허브 형태가 요즘 SaaS/Dev 포털 패턴.[^7]
    - “Use Cases / Guide / Changelog / Community Q\&A” 같은 섹션을 별도 탭으로.
- 레이아웃 패턴:
    - 왼쪽 사이드바에 카테고리 트리, 오른쪽에 문서/글 본문 – 전형적인 개발자 포털 UX.[^7]
    - 상단에는 검색창(“제품명·기술 키워드로 검색”)을 고정.
- 커뮤니티 요소:
    - 댓글 대신 GitHub Discussions/Slack/Discord 연결, 혹은 간단한 Q\&A 게시판.
    - “Issue → 글/문서로 환류되는 구조”를 의도적으로 보여주는 로드맵/Changelog 섹션.[^7]


## 5. 비주얼·스타일 가이드 제안

- 컬러:
    - 메인 사이트의 기업 블루·네이비 계열을 유지하면서, 포인트로 라이밍린/퍼플 등 한두 가지만 추가.[^1]
    - 다크 배경 + 밝은 카드 블록(네오 브루탈리즘까지는 아니어도, 대비가 분명한 구조).[^4]
- 타이포그래피:
    - 한글: 고딕계열(예: Pretendard, SUIT) 기반, 헤드라인은 두껍게, 본문은 가독성 중심.
    - 코드 블럭은 모노스페이스 폰트로 명확하게 분리.
- 인터랙션:
    - 카드 호버 시 살짝 확대, 그림자·보더 컬러 변화 등 미세 애니메이션.[^3][^4]
    - 스크롤 시 섹션 페이드인, 헤더는 스크롤 축소형(Sticky + Height 줄이기).

***

원하시면:

- Figma용 와이어프레임 구조(섹션별 레이아웃 명세)
- 카테고리 체계(예: PII/비식별, SIEM, NDR, DB, 클라우드)
- 실제 첫 화면 카피/섹션 문구 초안
까지 기술블로그 전용 IA/UX 문서 형태로 정리해 드리겠습니다.
