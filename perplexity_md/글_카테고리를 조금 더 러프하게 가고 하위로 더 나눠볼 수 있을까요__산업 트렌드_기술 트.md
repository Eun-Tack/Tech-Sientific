산업 트렌드
기술 트렌드
제품 리서치
거버넌스

월례회는 없애고
그냥 커뮤니티하고 해서 문서/가이드/사진

이렇게 가면 좋을 것 같아요

네, **더 러프하고 확장성 있는 구조**로 재설계했습니다. 산업/기술 트렌드 중심으로 가면서 하위 카테고리 풍부하게 구성했습니다.[^1][^2]

## 1. 수정된 메뉴 구조 (한국어, 러프한 느낌)

```
메인 네비게이션 (간결 6개 메뉴)
┌─────────────────────────────────────┐
│ [티사이언티픽 기술랩]  🔍 검색  🌙 다크 │
├─ 글 ─────────────────────────────────┤
│ ├─ 산업 트렌드                      │
│ ├─ 기술 트렌드                      │
│ ├─ 제품 리서치                      │
│ └─ 거버넌스                         │
├─ 커뮤니티 ──────────────────────────┤
│ ├─ 문서·가이드                      │
│ ├─ Q&A                              │
│ └─ 사진 갤러리                      │
├─ 소개 ──────────────────────────────┤
└─ 문의 ──────────────────────────────┘
```


## 2. '글' 메뉴의 러프한 하위 카테고리 상세

| 1순위 카테고리 | 하위 카테고리 예시 | 콘텐츠 예시 |
| :-- | :-- | :-- |
| **산업 트렌드** | 공공/금융, 제조/헬스케어, 조달청 동향, 해외 사례 | "2026 조달청 클라우드 보안 조달 트렌드", "금융권 SIEM 도입 사례" |
| **기술 트렌드** | 위협 인텔, 클라우드 보안, AI 보안, 제로트러스트 | "NDR vs SIEM 최신 비교", "GenAI 시대 PII 탐지 트렌드" |
| **제품 리서치** | i-PMS/P3RAI, SIEM/BigEye, NDR/PacketCyber, InfraEye | "i-PMS 룰 커스터마이징 가이드", "PacketCyber 네트워크 분석 PoC" |
| **거버넌스** | 개인정보, 규제 대응, 데이터 거버넌스, 컴플라이언스 | "P3RAI 비식별화 검증 방법", "K-ISMS-P 인증 대응 체크리스트" |

## 3. '커뮤니티' 메뉴 상세 (월례회 → 사진 갤러리 통합)

| 서브메뉴 | 페이지 내용 | 콘텐츠 예시 |
| :-- | :-- | :-- |
| **문서·가이드** | 제품 매뉴얼, API 레퍼런스, 핸즈온 | "InfraEye 설치 가이드", "BigEye 대시보드 커스터마이징" |
| **Q\&A** | 자주 묻는 질문, 기술 토론 | "SIEM 로그 연동 FAQ", "PII 마스킹 최적화 질문" |
| **사진 갤러리** | 월례회 사진, 팀 빌딩, 컨퍼런스 | "Tech 랩 월례회 \#01", "K-SEC 컨퍼런스 발표 사진" |

```

├─ 글 ──────────(아코디언: 산업/기술/제품/거버넌스)

```


## 7. 구글 안티그래비티 구현

```
컬렉션 구조:
1. Posts 컬렉션 → 필터: 산업트렌드/기술트렌드/제품리서치/거버넌스
2. Community 컬렉션 → 서브: Docs/QA/Gallery
3. Gallery → 이미지 중심 페이지 (라이브러리 그리드)

네비: "드롭다운 메뉴" → '글'과 '커뮤니티'에만 서브메뉴
```
